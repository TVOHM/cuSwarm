What Is cuSwarm?

cuSwarm is a library written for NVIDIA CUDA that is designed to simulate large numbers of swarming, flocking or schooling agents for use in real time games.

The agents are modeled with 3 behaviours that cause swarming behaviour:

Cohesion - Steers towards the perceived center of neighbours

Alignment - Steers towards the perceived heading of neighbours

Separation - Steers to avoid collisions with nearby neighbours

What Are the Requirements For Using cuSwarm?

An NVIDIA CUDA compatible graphics card - https://developer.nvidia.com/cuda-gpus

An IDE that supports the CUDA SDK, such as Visual Studio 2010 Express - http://www.microsoft.com/visualstudio/express

The CUDA SDK - https://developer.nvidia.com/cuda-downloads

It is also recommended that the latest device drivers are installed for your graphics card - http://www.nvidia.co.uk/page/drivers.html

How Do I Integrate cuSwarm With My CUDA Project?

Download latest version of cuSwam - https://github.com/TVOHM/cuSwarm (NOTE - Public repository will be empty until project is graded)

Place the following files from the latest version of cuSwarm into your project:

cutil_math.cuh, calculate_position.cuh, calculate_steering.cuh, config.cuh, make_rand_position.h, make_rand_velocity.h, simulation.cu, simulation.cuh

Include the header simulation.cuh to access the simulated swarm class.

How Do I Simulate A Swarm In My Game?

First instantiate a new simulation object.

simulation my_simulation;

Create a config object.

config my_config;

Modify the public variables to adjust the behaviour of the swarm:

config.num_boids, config.max_position, config.max_velocity, config.cell_dim, config.speed, config.cohesion_radius, config.cohesion_weight, config.alignment_radius, config.alignment_weight, config.separation_radius, config.separation_weight, config.skipthink_denominator

Apply the config object to your simulation using simulation.setConfig(). You may apply configurations at any time to a simulation in runtime.

my_simulation.setConfig(my_config);

Call simulation.update() whenever you need to update the swarm, passing the elapsed time in milliseconds since this function was last called.

my_simulation.update(123);

Use simulation.getPositions() and simulation.getVelocities() to get the current position, speed and heading of all agents for drawing.
